The cache section enables the BuildBuddy cache and configures how and where it will store data.

<Note>
This section is **optional**.
</Note>

## Configuration Options

### Common Options

<ParamField path="max_size_bytes" type="integer">
  How big to allow the cache to be (in bytes).
</ParamField>

<ParamField path="in_memory" type="boolean">
  Whether or not to use the in_memory cache.
</ParamField>

<ParamField path="zstd_transcoding_enabled" type="boolean">
  Whether or not to enable cache compression capabilities. You need to use `--experimental_remote_cache_compression` to activate it on your build.
</ParamField>

### Disk Cache

<ParamField path="disk" type="object">
  The Disk section configures a disk-based cache.
  
  <ParamField path="disk.root_directory" type="string" required>
    The root directory to store cache data in, if using the disk cache. This directory must be readable and writable by the BuildBuddy process. The directory will be created if it does not exist.
  </ParamField>
</ParamField>

### Enterprise-Only Options

<ParamField path="redis_target" type="string">
  A redis target for improved RBE performance.
  
  <Note>
    **Enterprise only**
  </Note>
</ParamField>

<ParamField path="gcs" type="object">
  The GCS section configures Google Cloud Storage based blob storage.
  
  <Note>
    **Enterprise only**
  </Note>
  
  <ParamField path="gcs.bucket" type="string" required>
    The name of the GCS bucket to store files in. Will be created if it does not already exist.
  </ParamField>
  
  <ParamField path="gcs.credentials_file" type="string" required>
    A path to a [JSON credentials file](https://cloud.google.com/docs/authentication/getting-started) that will be used to authenticate to GCS.
  </ParamField>
  
  <ParamField path="gcs.project_id" type="string" required>
    The Google Cloud project ID of the project owning the above credentials and GCS bucket.
  </ParamField>
  
  <ParamField path="gcs.ttl_days" type="integer">
    The period after which cache files should be TTLd. Disabled if 0.
  </ParamField>
</ParamField>

<ParamField path="s3" type="object">
  The AWS section configures AWS S3 storage.
  
  <Note>
    **Enterprise only**
  </Note>
  
  <ParamField path="s3.region" type="string" required>
    The AWS region
  </ParamField>
  
  <ParamField path="s3.bucket" type="string" required>
    The AWS S3 bucket (will be created automatically)
  </ParamField>
  
  <ParamField path="s3.credentials_profile" type="string">
    If a profile other than default is chosen, use that one.
  </ParamField>
  
  <ParamField path="s3.ttl_days" type="integer">
    The period after which cache files should be TTLd. Disabled if 0.
  </ParamField>
  
  <Note>
    By default, the S3 blobstore will rely on environment variables, shared credentials, or IAM roles. See [AWS Go SDK docs](https://docs.aws.amazon.com/sdk-for-go/v1/developer-guide/configuring-sdk.html#specifying-credentials) for more information.
  </Note>
</ParamField>

## Example Configurations

<CodeGroup>

```yaml Disk
cache:
  max_size_bytes: 10000000000 # 10 GB
  disk:
    root_directory: /tmp/buildbuddy-cache
```

```yaml GCS & Redis (Enterprise)
cache:
  redis_target: "my-redis.local:6379"
  gcs:
    bucket: "buildbuddy_blobs"
    project_id: "my-cool-project"
    credentials_file: "enterprise/config/my-cool-project-7a9d15f66e69.json"
    ttl_days: 30
```

```yaml S3 (Enterprise)
cache:
  s3:
    # required
    region: "us-west-2"
    bucket: "buildbuddy-bucket"
    # optional
    credentials_profile: "other-profile"
    ttl_days: 30
```

```yaml Minio (Enterprise)
cache:
  s3:
    static_credentials_id: "YOUR_MINIO_ACCESS_KEY"
    static_credentials_secret: "YOUR_MINIO_SECRET"
    endpoint: "http://localhost:9000"
    s3_force_path_style: true
    region: "us-east-1"
    bucket: "buildbuddy-cache-bucket"
```

</CodeGroup>
