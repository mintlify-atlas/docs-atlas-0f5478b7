In order to configure BuildBuddy's GitHub integration, you'll either need to:

- [create an access token](https://docs.github.com/en/github/authenticating-to-github/creating-a-personal-access-token) with the `repo:status` scope. This is the supported method for BuildBuddy open source.
- or [create a GitHub OAuth app](https://docs.github.com/en/developers/apps/creating-an-oauth-app). In the `Authorization callback URL` field - you'll need to enter your BuildBuddy application url, followed by the path `/auth/`. For example: `https://app.buildbuddy.io/auth/`. This is the recommended method for BuildBuddy Enterprise.

<Note>
This section is **optional**.
</Note>

## Configuration Options

The GitHub section enables the posting of BuildBuddy GitHub commit statuses for CI runs.

<ParamField path="access_token" type="string">
  The GitHub access token used to post GitHub commit statuses.
</ParamField>

<ParamField path="client_id" type="string">
  The client ID of your GitHub OAuth App.
  
  <Note>
    **Enterprise only**
  </Note>
</ParamField>

<ParamField path="client_secret" type="string">
  The client secret of your GitHub OAuth App.
  
  <Note>
    **Enterprise only**
  </Note>
</ParamField>

## Example Configurations

<CodeGroup>

```yaml Open Source (Access Token)
github:
  access_token: abc123
```

```yaml Enterprise (OAuth App)
github:
  client_id: abc123
  client_secret: def456
```

</CodeGroup>
