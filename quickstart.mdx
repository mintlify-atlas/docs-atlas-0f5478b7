BuildBuddy is a fully managed cloud Bazel platform. It provides a results store & UI, remote build caching, remote build execution, and more.

It's easy to get set up and is **free for individuals and open source projects**. For companies, we offer an [Enterprise](/enterprise) version of BuildBuddy that contains advanced features like OIDC Auth, API access, and more.

## Setup

<Steps>
  <Step title="Add BuildBuddy to your .bazelrc">
    To use BuildBuddy's Results UI, you just need to configure Bazel to send build events to our cloud BuildBuddy instance. The easiest way to do this is with a `.bazelrc` file in the root of your project.

    ```bash .bazelrc
    build --bes_results_url=https://app.buildbuddy.io/invocation/
    build --bes_backend=grpcs://remote.buildbuddy.io
    ```

    That's it, 2 lines and you're up and running!
  </Step>

  <Step title="Run a Bazel build">
    Now, when you build or test with Bazel, it will print a URL where you can view your build or test results:

    ```shellsession
    tylerw@lunchbox:~/buildbuddy-io/buildbuddy$ bazel build server:all
    INFO: Streaming build results to: https://app.buildbuddy.io/invocation/24a37b8f-4cf2-4909-9522-3cc91d2ebfc4
    INFO: Analyzed 13 targets (0 packages loaded, 0 targets configured).
    INFO: Found 13 targets...
    INFO: Elapsed time: 0.212s, Critical Path: 0.01s
    INFO: 0 processes.
    INFO: Streaming build results to: https://app.buildbuddy.io/invocation/24a37b8f-4cf2-4909-9522-3cc91d2ebfc4
    INFO: Build completed successfully, 1 total action
    ```

    You can ⌘ + click on these URLs to quickly view the invocation's details.
  </Step>
</Steps>

<Tip>
For more advanced configurations, see the [Authentication](#authentication) and [More features](#more-features) sections below.
</Tip>

## Authentication

BuildBuddy offers two authentication options which are easy to configure:

<AccordionGroup>
  <Accordion title="Unauthenticated (Public)">
    Your build logs are uploaded over an encrypted gRPCS/TLS connection and will be accessible with anyone you share your BuildBuddy URL with, without credentials. They will not be associated with any BuildBuddy account or organization.

    **Best for:**
    - Quick testing
    - Public open source projects
    - Non-sensitive builds
  </Accordion>

  <Accordion title="API Key Authentication (Recommended)">
    Your build logs are uploaded over an encrypted gRPCS/TLS connection, and will be associated with your account. Only your account and members of your BuildBuddy organization will be able to view your build logs.

    **To configure:**
    1. [Create a BuildBuddy account](https://app.buildbuddy.io/)
    2. Visit the [Quickstart page](https://app.buildbuddy.io/docs/setup) which will now contain authentication options
    3. Add your API key to your `.bazelrc` file

    **Best for:**
    - Team collaboration
    - Private repositories
    - Enterprise use cases

    For more information see our [Authentication Guide](/guide-auth).
  </Accordion>
</AccordionGroup>
